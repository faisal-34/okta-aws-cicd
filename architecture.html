Okta AWS CI/CD Pipeline Design overview — Identity-integrated infrastructure pipeline using GitHub Actions + Terraform + AWS + Okta

Business Context

Objective

Automate provisioning of AWS infrastructure and Okta identity configuration from a single Git push.

Users

Developers push code; Okta SAML SSO grants AWS Console access to engineers via federated roles.

Value

Eliminate manual infra changes. Single source of truth for cloud resources and identity policies.

Compliance

All infrastructure changes tracked in Git history. IAM roles scoped to least-privilege ECR read.

Features

🔁 Automated CI/CD

  • Push to master triggers full pipeline
  • Build → Terraform → Deploy in sequence
  • 15-minute job timeout prevents hangs
  • Job outputs pass EC2 IP between jobs

🏗 Infrastructure as Code

  • EC2 with correct VPC & subnet
  • Security group scoped to VPC
  • IAM instance profile for ECR access
  • State stored in S3 for team sharing

🔐 Identity & Access

  • Okta SAML SSO to AWS Console
  • Federated Admin & ReadOnly roles
  • EC2 IAM role (ECR read only)
  • No static credentials on EC2

📦 Container Delivery

  • Docker image per git commit (SHA tag)
  • Private ECR registry per AWS account
  • Zero-downtime swap (stop → rm → run)
  • --restart always for auto-recovery

🧩 Modular Terraform

  • terraform/aws/ — AWS resources
  • terraform/okta/ — Okta (manual, one-time)
  • Conditional Okta resources (count = 0 if no metadata)
  • Backend isolated from provider config

🔒 Security Posture

  • Secrets in GitHub encrypted store only
  • Okta provider excluded from CI (no token in runner)
  • SG restricts inbound to ports 22, 80, 3000
  • Least-privilege ECR read policy on EC2

Deployment Model

Trigger

Any git push to master or main. All 3 jobs run in sequence with dependency chain.

State Management

Terraform state in S3 bucket okta-cicd-tfstate-<account-id>. Enables incremental applies — only changed resources are updated.

EC2 Lifecycle

Terraform manages EC2 lifecycle. Adding iam_instance_profile or associate_public_ip_address forces recreate (immutable fields).

Okta SSO Setup

One-time manual run of terraform/okta/ with OKTA_API_TOKEN. Not part of CI. Creates SAML IdP + federated IAM roles.
